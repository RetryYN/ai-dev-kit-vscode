## 3. 発見した差異と対処

### 差異1: モデル割当ルール（影響度: A - 重大）

**企画書の想定**（Sonnet中心）:
```
| 難易度スコア | 担当モデル |
| 0-3点 | Haiku |
| 4-7点 | Sonnet |
| 8-10点 | Sonnet + Codex補助 |
| 11-14点 | Sonnet（Opus初期設計） |
```

**実装の実態**（Codex中心）:
```
| スコア | モデル |
| 0-3 | Haiku 4.5 |
| 4-7 | Codex 5.2 |
| 8-10 | Codex 5.3 |
| 11-14 | Codex 5.3（Opus が初期設計） |
```

**根本原因**: 企画書作成時はSonnet 5.0を想定していたが、その後Codex採用に移行。企画書は更新されずアーカイブ化された。

**対処内容**:
- 企画書にアーカイブ資料であることを明示する注意書きを追加
- 現行仕様（SKILL_MAP.md + CLAUDE.md）への導線を明記
- MEMORY.md に「spec is reference-only」と記録済みであることを確認

**対処結果**: ✓ 完了（2026-02-11実施）
